# Redis State Storage

By default, Telegrator uses an in-memory storage for user states. This is great for development but not suitable for production where bot restarts would lose all user progress. The `Telegrator.RedisStateStorage` package provides a persistent alternative.

## Installation

```shell
dotnet add package Telegrator.RedisStateStorage
```

## Configuration

To use Redis storage, you need to register it in your application's DI container.

### Using Generic Host

```csharp
using Telegrator.Hosting;
using Telegrator.States;
using StackExchange.Redis;

var builder = Host.CreateApplicationBuilder(args);

// 1. Connect to Redis
var redis = ConnectionMultiplexer.Connect("localhost:6379");

// 2. Add Telegrator
builder.AddTelegrator();

// 3. Replace default storage with Redis
builder.Services.AddStateStorage<RedisStateStorage>();
builder.Services.AddSingleton<IConnectionMultiplexer>(redis);

var host = builder.Build();
host.UseTelegrator();
await host.RunAsync();
```

## How it Works

The `RedisStateStorage` automatically serializes your state objects to JSON and stores them in Redis using the keys generated by your `IStateKeyResolver` (e.g., `SenderIdResolver`).

- **Key Format**: `[TypeName]:[ResolvedUID]` (e.g., `UserState:123456789`)
- **Persistence**: States survive bot restarts and can be shared across multiple instances of your bot.

> [!IMPORTANT]
> Ensure that your state types (enums or classes) are JSON-serializable.

## Sharing Redis with Multiple Bots

If you host multiple bots on the same Redis server, ensure they don't collide. You can use different Redis databases or prefix your keys in a custom `IStateStorage` implementation if needed.
